## Overview

Agentic artifact surfaces are places where the agent can create, inspect, revise, and operate on the actual work product rather than only talking about it. Jason Liu's Codex-maxxing article frames the side panel as the place where Codex stops being only chat and starts becoming where the work happens. HTML Anything and Presenton are adjacent examples: agent-era tools where the output is HTML, slides, images, or presentations rather than a plain text answer.

The durable idea is that knowledge work becomes easier to delegate when the artifact is inspectable in the same loop as the agent's actions.

Thariq's HTML article makes the strongest case: HTML is denser than markdown for tables, CSS, SVG, code snippets, interactions, workflows, spatial layouts, images, and shareable specs. Dave Jeffery's workflow prompt adds a specific form: one HTML page plus a JSON data file that documents app flows and lets a human click an action to highlight how packages/components interact.

The 2026-06-11 captures broaden the artifact surface beyond static reports. OpenScreen is a free/open-source Screen Studio alternative for polished screen demos and walkthroughs. Samuel Spitz's Replit Slides post reinforces that AI slide quality depends on the underlying code/design representation, not only prompt wording. Oliver Kenyon's post is a reminder that AI-generated marketing images and flyers become more useful when they remain editable. Dominik Kundel's Build Web Apps article points to a design-to-implementation loop: generate a UI concept with image generation, implement it in Codex, then compare screenshots/designs to preserve intent.

The video-editing tutorial transcript adds a concrete coded-video workflow. Codex can edit a Remotion project, add a title sequence, generate or insert music, iterate on visual timing/text, and render the final video. Seth's note suggests trying Hyperframes instead of Remotion, so the reusable workflow should be engine-agnostic: script/storyboard, coded timeline, asset generation/import, preview loop, critique, and final render.

The video-use capture makes the agentic editing workflow much more concrete for founder/product demo collateral. It treats video as something the agent can read through transcript packs and on-demand timeline composites rather than full-frame dumping. The repo's stated loop is: drop raw footage into a folder, let the agent inventory and propose an editing strategy, confirm, then produce `edit/final.mp4`. Its primitives map well to Seth's job-application use case: remove filler words and dead space, color-grade segments, add short audio fades, burn styled subtitles, generate animation overlays via HyperFrames/Remotion/Manim/PIL, self-evaluate rendered cut boundaries, and persist project memory for follow-up sessions.

Matt Van Horn's Remotion-vs-HyperFrames article adds the missing routing layer. His conclusion is not to pick one winner: Remotion remains the mature React/code-review/Lambda-scale renderer, while HyperFrames is the faster agent-native HTML route for one-off launch reels, product demos, captioned clips, and weekly explainers. The shared lesson is that good output comes from structure-first prompting, iteration, explicit pacing, and reusable constants/templates. For Seth's product-demo collateral, this reinforces a practical stack: keep Remotion, HyperFrames, and video-use available; route talking-head/product-demo edits through transcript-first cutting, then choose HyperFrames for fast HTML-based overlays or Remotion for maintainable React compositions and scale.

The Acme bundle adds a brand-machine angle: brand guideline -> design system -> website refresh -> calculators -> API-demo artifact repo -> launch videos -> account decks. Open Design is a particularly relevant lead because it packages `DESIGN.md`, skills, plugins, MCP, decks, prototypes, images, and HyperFrames into an agent-native design surface that can work with Codex/Claude/Cursor.

Frontend Slides is a deck-specific skill lead for the same artifact lane. It creates fixed 16:9 single-file HTML presentations with inline CSS/JS, can convert PowerPoint content into web slides while preserving assets, and uses visual style discovery: generate several title-slide previews, let the human pick a direction, then expand only the selected style/template. The key design pattern is progressive context loading: read a compact template index first, load preview cards for shortlisted candidates, and load the full design spec only after selection.

The Desktop Archive artifact notes add more follow-up leads: Figma MCP workflows, Excalidraw and draw.io MCPs, Remotion product demo recipes, Google VEO/watch items, Pencil downloads, TikTok-style slide shows, and another Frontend Slides pointer. These should be treated as an artifact-tool evaluation queue, with each promising tool captured separately before becoming a recommendation.

The Sunder sync adds recovered artifact and website-design leads from the older workspaces: HTML-as-interface notes, slide/deck tools, design-system references, website revamp material, and screen/video automation ideas. Keep the digest as the router and promote only the rows with complete raw captures or locally preserved scrape files.

## Key Ideas

- Side-panel review matters because the user and agent can look at the same artifact: markdown, spreadsheets, PDFs, slides, local web pages, Streamlit apps, Jupyter notebooks, or static HTML.
- HTML can be a first-class output format for agent-created deliverables because humans often want a rendered page, report, card, or deck more than markdown.
- Agentic tools are converging on reusable surface templates: articles, keynote decks, resumes, posters, tweet cards, web prototypes, data reports, and videos.
- Presentations are a recurring artifact class. Presenton is a self-hostable AI presentation generator/API with model and data control, which makes it relevant for agent-generated decks.
- The review loop is the product: generate, inspect visually, annotate, revise, export, and preserve the source artifact.
- Cross-harness artifact libraries matter because the same output patterns can work across Claude Code, Claude Desktop, Codex, Gemini CLI, OpenClaw, and Hermes.
- Diagram generation is a distinct artifact lane. `fireworks-tech-graph` turns natural-language system descriptions into SVG/PNG diagrams, which may be better than raw Excalidraw for repeatable technical diagrams.
- PDF redesign is another artifact lane: agents can transform existing resumes, invoices, reports, and proposals into more polished documents, but visual verification matters.
- Screen recordings are artifacts too. A useful agent-era screen tool should support capture, auto-zoom, cursor polish, captions, annotations, trimming, and export without forcing manual editing for every demo.
- AI slides and AI flyers should be evaluated by editability and export fidelity, not just first-generation aesthetics.
- Website revamps can use a richer loop: generate a design image, implement the web app, run browser validation, and do a side-by-side critique against the target design or screenshot.
- Video generation should be treated as a coded artifact workflow, not a one-shot render: keep scenes/timing/assets in source, preview frequently, then render only after visual checks.
- Remotion and Hyperframes should be compared on the same workflow: setup friction, agent-editable timeline model, preview speed, asset handling, caption/audio support, and final export quality.
- Transcript-first video editing may be the strongest practical interface for talking-head product demos. The agent can reason over word-level timestamps and call visual timeline views only at ambiguous cuts, which keeps the editing loop inspectable and cheaper than feeding every frame to a model.
- For founder-narrated collateral, the likely stack is raw takes plus product footage -> transcript pack -> EDL/cut plan -> FFmpeg render -> subtitles/color/audio fades -> HyperFrames/Remotion overlays -> rendered-boundary self-eval -> final MP4.
- The video renderer should be selected per job. HyperFrames fits fast agent-authored HTML launch reels and captioned product clips; Remotion fits React-native compositions, code-reviewed templates, and large-scale rendering; video-use can sit upstream as the transcript-first editing layer for raw talking-head footage.
- The quality ceiling is pacing and structure, not just renderer choice. Useful prompts specify scene structure, duration, resolution, FPS, editable constants/templates, sharp cuts, holds, fades, and validation before final render.
- A serious enterprise website refresh should start from brand guidelines and `DESIGN.md`, then use image-to-code or design-to-code loops to produce inspectable sections.
- Account-specific decks, calculators, and launch videos should be generated from the same brand system so GTM assets do not drift visually.
- Deck-generation skills should show visual previews before committing to a full deck. Style selection is easier when the human compares rendered options instead of describing taste in words.
- Single-file HTML decks are a strong agent artifact format when the goal is quick sharing, browser preview, source editability, and no build-tool dependency.
- Diagram and design MCPs belong in the same artifact taxonomy as slides/videos because the user-facing output is visual, inspectable, and often needs export fidelity.

## Open Questions

- Which artifact format should Seth standardize for reusable outbound assets: static HTML, Slidev, PPTX, or Presenton?
- Should Second Brain synthesis pages ever generate companion HTML reports or decks?
- What verification gate catches broken layouts: screenshots, browser automation, export checks, or manual review?
- Should Seth try OpenScreen as the default free Screen Studio replacement for demo capture?
- For website revamps, should Codex start from GPT-image design concepts, reference screenshots, raw HTML prototypes, or a written requirements doc?
- Should the upgraded video workflow standardize on video-use as the upstream editing layer, with HyperFrames or Remotion selected per job?
- Should Seth trial video-use directly for the job-application product collateral, using one short raw talking-head take and one product screen recording as the first benchmark?
- For Acme, should Open Design be the brand/design operating surface, or should Codex Build Web Apps plus a hand-authored `DESIGN.md` be enough?
- Should Frontend Slides become the default path for quick HTML decks, while PPTX remains only for final customer-facing exports?
- Which visual artifact tool should be trialed first from the Archive: Figma MCP, Excalidraw MCP, draw.io MCP, Pencil, VEO, or a Remotion product-demo template?
