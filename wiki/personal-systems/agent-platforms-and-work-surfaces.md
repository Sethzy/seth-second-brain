## Overview

This page tracks agent platforms and work surfaces that might shape Seth's personal operating system. The theme is not one winning app; it is where agents should live: Slack, a local markdown vault, a browser/computer, a cloud computer, a company search/RAG layer, an email client, or a launcher-style operating surface.

Hermes, OpenClaw/Clawchief, Junior, CodeRabbit Agent, Onyx, SimpleWords, Tolaria, and Raycast all point at the same direction: agents become useful when embedded in the places work already arrives and when they can connect tools, memory, approvals, and audit trails. Clawchief v2 is especially relevant because it moved from a prompt-heavy assistant into a more legible operating layer: source-of-truth files for priorities/tasks/meeting notes/action policy, a live task file plus archive, a heartbeat orchestrator, cleaner cron templates, and skills for executive-assistant, business-development, daily task management, and daily prep.

Ryan Carson's original OpenClaw assistant post clarifies the applied CRM/outreach shape of Clawchief: the agent watches inbox and calendar, parses booking links, follows up on unreplied emails, prepares task lists, avoids duplicate tasks, updates an outreach tracker or CRM from email activity, researches suppliers/partners, and sends only high-signal updates. The important architecture is durable context in files plus Gmail/Calendar/Sheets, with cron sweeps making the assistant proactive and a validation checklist proving it can search mail, update the tracker, promote due tasks, archive completions, and ingest meeting notes.

Jess Yan's managed-agents note adds a product-management angle: the product role changes when DIY agents can absorb research, context gathering, drafting, and recurring operational work. The important question becomes which workflows deserve agent ownership and how the product/human role monitors outcomes.

Scout adds a more concrete company-brain architecture: live context providers over Slack, Drive, CRM, knowledge wiki, voice wiki, workspace, and MCP servers; a CRM/wiki that accumulates learned facts; and scheduled follow-ups. Its sharpest design claim is "navigation over search": walk live sources and preserve real citations instead of relying only on stale vector chunks.

The Acme bundle and Dan Rosenthal post make the same architecture operational for a GTM team: put the company brain in markdown/GitHub, sync Slack/calls/Drive/campaign data into account repos, expose tools through MCPs, and make skills/playbooks available to every teammate. For Acme, the first concrete product should likely be a second-brain Slackbot that sales and tech both trust.

The Vercel Slack Agent Template is a concrete implementation lead for that Slackbot surface: Slack Assistant events and thread context start a durable workflow, AI SDK tools run inside a `DurableAgent`, responses stream back into Slack, and Workflow DevKit hooks pause sensitive actions for approval.

Kyle Jeong's Browserbase `bb` article is the strongest current replacement candidate for the earlier Viktor.ai question. Instead of a narrow company-brain app, `bb` is a generalized Slack-native internal agent: one core agent loop, isolated cloud sandboxes, browser automation for human-only web apps, skills/playbooks loaded on demand, scoped permission sessions, credential brokering through a proxy, and thread-persistent Slack workspaces.

10xapp Core OSS is a different kind of work-surface lead: an open-source all-in-one productivity app rather than an agent wrapper. Its useful primitives are Gmail/Outlook sync, Google/Microsoft calendar sync, AI chat with tool use, real-time messaging, files, collaborative documents, projects, workspaces/RBAC, and a unified dashboard. The stack is also agent-friendly for local hacking: FastAPI, React/Vite, Supabase/Postgres/RLS/Realtime, Cloudflare R2, OAuth, and swappable AI/search/queue/analytics providers.

The Devin/Ramp pair sharpens the platform decision. Ryan Carson's Devin note argues for buying the managed code-factory platform when the goal is simply to ship faster: his Linear `land` playbook runs CI and merges green PRs, and his conclusion is "plug into the wall" instead of rebuilding the electricity company. Ramp Inspect shows when internal ownership can still be worth it: a Slack/web/Chrome/PR/VScode multiplayer coding surface where sessions run in prepared cloud sandboxes, carry Ramp-specific context, and verify work like a Ramp engineer.

Cognition's Devin articles add the platform details behind that buy path. Internally, Cognition uses Devin across web, Slack, Linear/Jira, CLI, API, codebase Q&A, Devin Review, DeepWiki, DANA, playbooks, MCPs, session insights, and automated triggers. The practical platform lesson is that managed agents become valuable when they are embedded where work arrives, can access the right repo/tool context, can review and autofix mechanical issues, and can turn repeated wins into playbooks.

Aaron Levie's capability-overhang article explains why the same agent platform shift is slower outside coding. Coding has unusually dense context inside the repo; enterprise knowledge work depends on fragmented systems, meetings, external events, legacy/on-prem stores, and complex permission boundaries. The winning work surface may be the one that makes the organizational brain legible, permissioned, and manageable for nontechnical users while also handling change management through FDEs, SIs, and internal operators.

The Desktop Archive platform notes widen the watchlist with OpenClaw-adjacent systems and surfaces: Composio secure OpenClaw, LarrySkill, DenchClaw, CoWork-OS, Voicebox, Junior, ChatAvocado, Core, OpenViktor, Hermes/OpenClaw setup notes, Telegram wrappers, and a large "cute idea for my app" capture around copywriting/product inspiration. Most are link leads, so the right next step is selective source capture before elevating claims.

The Sunder sync adds a larger recovered source lane for platform experiments: OpenClaw/Clawchief setup notes, Viktor/Tasklet-style work-surface leads, browser-agent and company-brain captures, Slack/agent template material, and older local scrape outputs. Use the sync digest as the router and then open the raw capture before deciding whether a platform deserves promotion into an active recommendation.

## Key Ideas

- Clawchief is an OpenClaw starter kit for a founder/chief-of-staff OS with priority map, auto-resolver, meeting-note ingestion policy, canonical live tasks, completed-task archive, environment notes, heartbeat orchestration, cron templates, and skills.
- The OpenClaw CRM-assistant pattern treats the tracker/sheet as a source of truth: email and calendar events become task, follow-up, and CRM updates only when the assistant can leave an audit trail.
- Hermes leads emphasize model routing, preference learning, context switching, and integrations such as Firecrawl, Browserbase, Google Workspace, GitHub, Stripe, Obsidian, Readwise, Granola, and Apify.
- Managed-agent businesses package agents as maintained digital employees: infrastructure, security, setup, watchdogs, updates, and customer-facing project management.
- Slack-native agents are a strong pattern. Junior and CodeRabbit Agent both position Slack as a runtime for investigation, context gathering, and action across connected tools.
- Vercel's Slack template turns that pattern into an implementation skeleton: Slack Bolt, AI SDK tools, AI Gateway, Workflow DevKit durable execution, and approval hooks.
- Browserbase's `bb` pattern suggests one generalized internal teammate can beat a fleet of narrow bots if the abstractions are right: sandbox, proxy, harness, interface, skills, and scoped permissions.
- Onyx is a company knowledge/RAG surface with connectors, custom agents, actions/MCP, web search, code execution, desktop/browser extensions, and chat/search.
- Tolaria is a local, git-first markdown knowledge-base app for second brains, company docs, and assistant memory/procedures.
- SimpleWords is an email/writing lead to evaluate against Superhuman-like workflows.
- Raycast's cross-platform rewrite is a work-surface lead: launcher, extensions, notes, native speed, and a place where AI can become ambient.
- 10xapp Core OSS is a candidate open-source substrate for an internal productivity/agent surface because email, calendar, messages, docs, files, projects, workspace permissions, and dashboard already live in one app.
- Devin is a managed-platform candidate for teams that want code-factory outcomes without owning orchestration infrastructure.
- Devin's current platform shape includes Slack/Linear/Jira/web/CLI/API entry points, codebase Q&A, PR review, DeepWiki, playbooks, MCPs, session insights, and autofix loops.
- Ramp Inspect is the custom-platform exemplar: many clients on one sandbox/API core, repository routing, multiplayer sessions, visual verification, and merged-PR metrics.
- Managed-agent workflows should be evaluated by the product outcome they unlock, not by agent novelty: time saved, decisions improved, artifacts shipped, or operations kept current.
- Company-brain systems need both live context access and durable compiled memory: query the current source when truth lives there, and write CRM/wiki records when facts need a home.
- Enterprise agent platforms win by turning messy organizational context into legible, permission-aware work surfaces that nontechnical managers can supervise.
- Capability is not the same as diffusion. Workflows, data formats, governance, literacy, permissions, and change management determine how fast agents actually reach legal, finance, consulting, manufacturing, and GTM teams.
- Context-provider architectures can make multi-source agents more reliable by keeping each source's quirks outside the main agent's prompt/tool history.
- A company OS needs contribution loops, not just search: Slack capture, account updates, call notes, customer pain points, and corrected agent outputs should flow back into durable memory.
- Safeguard files and approval rules are part of the platform, especially when MCP tools can act in CRM, email, outreach, or data systems.
- Platform watchlists should separate "try this product/repo" from "adopt this architecture." The Archive batch is mostly a try/evaluate queue.

## Open Questions

- Should Seth's next OS experiment be Hermes, Clawchief/OpenClaw, Tolaria, Onyx, or Raycast?
- Should Clawchief's v2 file layout become the template for Seth's own personal ops layer: `priority-map.md`, `auto-resolver.md`, `meeting-notes.md`, `tasks.md`, `tasks-completed.md`, `TOOLS.md`, and `HEARTBEAT.md`?
- Should the OpenClaw CRM-assistant pattern be adapted for Acme/eGiro first around Gmail, Calendar, Sheets/Attio, and a narrow outreach tracker before trying a full company OS?
- Which surface should own recurring checks: Slack, Codex threads/goals, local cron, Hermes, or OpenClaw?
- Is SimpleWords actually better than Superhuman for Seth's email workflow, or just a separate writing layer?
- Which integrations are non-negotiable for Seth: Gmail, Calendar, Drive, GitHub, Slack, Obsidian/markdown vault, Stripe, browser, or Readwise?
- Which DIY managed agents would materially change Seth's product/research cadence rather than just adding another surface?
- Should Scout be tried as the local/company-brain baseline against Onyx, Tolaria, and this repo's existing LLM wiki pattern?
- Should Acme's Slackbot/company brain use the Browserbase `bb` pattern as the baseline instead of Viktor.ai: one Slack-native agent loop, sandboxed execution, skill routing, credential proxy, and hard-scoped permissions?
- Should the first Slackbot prototype use the Vercel Slack Agent Template, Scout, or a minimal local Slack app wired into this repo?
- Should 10xapp Core OSS be evaluated as a self-hostable company/productivity OS substrate, or is it too much application surface compared with a Slack-native agent plus this markdown wiki?
- For coding work, should Seth/Acme buy Devin/Codex-style managed orchestration first, or build only the smallest Inspect-like internal layer where proprietary context and verification matter?
- For Acme/company OS, which context should become legible first: CRM/account notes, Slack and calls, product/API docs, customer pain points, finance/ops workflows, or market-trigger feeds?
- Which OpenClaw-adjacent lead from the Archive should be evaluated first against the current Clawchief/Browserbase/Vercel/10xapp shortlist?
