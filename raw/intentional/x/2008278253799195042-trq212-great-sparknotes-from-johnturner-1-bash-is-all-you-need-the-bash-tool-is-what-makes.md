## Verbatim Text

Great sparknotes from @johnturner 

1. "Bash is all you need" — The bash tool is what makes Claude Code so powerful. It was the first "code mode" — composable, discoverable, and lets you use existing software (ffmpeg, jq, etc.) without building custom tools.

2. The agent loop has 3 parts: Gather context → Take action → Verify work. Verification is crucial — if you can verify the output, it's a great candidate for an agent.

3. Code generation for non-coding tasks — Ask Claude to find the weather? It might write a script to fetch a weather API, get your location dynamically, and compose multiple APIs together. Codegen = composing APIs.

4. Tools vs Bash vs Codegen tradeoffs:
- Tools: Reliable but high context usage, not composable
- Bash: Composable, low context, needs discovery time
- Codegen: Highly flexible, longest to execute

5. Skills are just folders — They're collections of files your agent can cd into and read. Very "file system pilled."

6. Security = Swiss cheese defense — Model alignment + harness permissioning + sandboxing. Multiple layers, none perfect alone.

7. The #1 metalearning: Read your agent's transcripts over and over. Understand what it's doing and why.

8. Every agent needs a container — File system + bash + ability to operate on it. Very different hosting architecture than traditional apps.
