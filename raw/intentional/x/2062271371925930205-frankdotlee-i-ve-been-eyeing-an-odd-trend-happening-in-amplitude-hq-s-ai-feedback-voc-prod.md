i’ve been eyeing an odd trend happening in @Amplitude_HQ's ai feedback (VoC) product the last few months… 👀👀

so I track WAU for a few new actions we’ve shipped in ai feedback. most have held steady, but one line has continued climbing every week: the number of users consuming insights and feedback data via our MCP server and agents (as opposed to in amplitude’s UI).

it started 2026 at 0 but now it’s running @ ~2-3x the rate of our next best action and seems pretty clear it’ll soon become the primary way ai feedback is used in the future. there’s been a ton of chatter recently that products need to go “headless” since agents will become primary consumers of products in the future, but it was fun seeing this exact thing happening in of our products!

fwiw, this didn't happen by itself… @spencerpauly and team shipped:

1. new mcp tools for each cut of the data - insights, feedback, mentions

2. built skills and agents using the tools, enabling teams to build automations, then made them accessible in plugins in @claudeai, @OpenAI codex, and @cursor_ai

3. iterated a ton on the names, descriptions, and overall quality of the MCP tools + agents/subagents we built so everything worked reliably

4. invested in adding tons of new feedback sources + tuned quality on our insights so customers would could trust and lean on our insights

shameless plug: If you haven't looked at amplitude ai feedback lately: we now ingest @SlackHQ, @meetgranola, @Gong_io, @Zendesk, @intercom, have a freeform API, and a bunch of other core feedback sources now. insight quality is way up, plus we’ve released specialist feedback agents that monitor and report on all of it for you, in or outside of amplitude.

hit me or @yanabana up if you’d to try this out or use ai feedback as new insight source for feeding what your software factories work on!! 😁
