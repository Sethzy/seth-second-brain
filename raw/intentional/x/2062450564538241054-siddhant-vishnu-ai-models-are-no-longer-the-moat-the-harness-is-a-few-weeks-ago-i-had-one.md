## Verbatim Text

AI models are no longer the moat. The harness is.

A few weeks ago, I had one of those moments where the future felt weirdly real. I was using Codex to makes videos from storyboarding to edits to final polished video.

Not prompt-to-video. Real editing. Source footage, timestamps, motion graphics, captions, audio, frame matching, revisions, final MP4. I would say things like: make the transition smoother, keep the logo until the caption exits, start the camera move 0.3 seconds earlier, end the video at exactly 24 seconds. And Codex would go change the actual production files, render again, and give me a new cut.

That is the part people are still underestimating.

The model is impressive, obviously. But the model alone is not the product. A raw model in a chat box can suggest, describe, and reason. It cannot reliably ship creative work unless it is inside a system that gives it tools, constraints, files, renderers, verification, and a workflow. That system is the harness.

For video, my harness is Codex + HyperFrames + FFmpeg + a repeatable video creation workflow. HyperFrames makes the core idea very simple: write HTML/CSS/GSAP, get an MP4.

Suddenly a reel is not an opaque timeline inside an editor. It is scenes, layers, assets, captions, keyframes, easing curves, audio tracks, and render commands. And because it is code, an agent can edit it.

That is the mental shift. Not “AI made a video.” AI operated the video creation pipeline.

I used this workflow for a Mirage promo video and a thumb-swipe video:

[promo video](https://www.instagram.com/p/DYuIBLWuV2o/)
 [thumb-swipe video](https://www.instagram.com/p/DY4qr_0o_h1/)

The workflow is surprisingly practical. First, I define the audience, format, goal, assets, timestamps, music, captions, CTA, and the emotion the video needs to create. Then Codex writes a second-by-second storyboard before touching the render.

Only after the plan is locked does it build the video in HyperFrames.

After that, the loop feels like editing with a very technical creative partner. I watch the render and give notes.

The transition is visible. The caption leaves too early. The logo needs to exit with the text. The camera should move slower. The final frame needs to match the screenshot exactly.

Then the agent adjusts the production system and renders again.
 
What I experienced with codex making the video was truly unreal and it is why I think the a lot of people are missing the point. Everyone is arguing about which model is better, but for many real workflows the bigger difference is the environment around the model. Can it see the files? Can it inspect outputs? Can it extract frames? Can it use FFmpeg? Can it render intermediate versions? Can it verify duration, resolution, audio, and frame timing? Can it follow a creative process instead of guessing?

If yes, the model becomes useful in a completely different way.

The harness gives the model hands.

Once the model has hands, creative software starts to look different. The future is not just better prompt boxes. It is agents operating inside deterministic production systems. The next AI winners will not simply be the people with access to the best model. Everyone will have good models. The winners will have the best harnesses: the best workflows, the best context, the best tools, the best taste encoded into repeatable systems, and the best verification loops.

I’m convinced code is the path to AGI.

Why?

Because code gives the model something reality can judge. Code is testable, cheap, easy to run and models are really good at it already.

Code can become a website, a video, an image pipeline, an automation, a UI, a simulation, a document, a render engine. It does not stop at screens. AI can already operate machines built and working in the real world. Motors, sensors, cameras, robots, browsers, APIs, renderers, tools.

The moment code becomes the interface, the model is not just talking. It is acting.

That power feels unreal. Almost magical.

So when I made a reel with Codex, I wasn’t really watching AI “generate a video.” I was watching AI write and operate a machine that produces video. That distinction feels massive.

Because if the model can use code to turn intention into any medium, operate tools in the real world, and verify the result, what exactly is left between “coding agent” and “creative agent”?

That is why this feels bigger than video. The reel was just the artifact. The real thing was the loop: intent → code → output → inspection → revision → shipped work.

So if you are building with agents, invest in the harness. The model is becoming the commodity. The harness is becoming the product.
