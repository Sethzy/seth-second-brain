Building on @gregisenberg’s latest post breakdown of AI Native Orgs: If People + Agents + Context is the strategy, then Game System Architecture is the engineering implementation.

In AI Native Organization, creating an agent is like designing and building a real game system.

**This post is from notes I'm compiling and brainstorming ideas for, written at 4 AM, feeling a bit groggy.**

Essentially, we're developing AI agents for organizations. This involves setting up systems and effective feedback loops to ship products quickly. This includes managing individual agents to match their skill sets—a bit like co-founder matching with agents (finding domain experts to manage agents in specific areas; this will lead to great success)—and writing contextual information from the know-how of each part of the organization, from build and sales to customer personas.

If these things are done well, the system will keep compounding until no one can surpass it.

The main operational loop that we should be working on from now on is:

Let's start with the people aspect, who manage the Strategy, Taste, and Trust.

What are your unique aspects of yourself? What are your specific interests? Just put them all in. Can be anime / game / rocket / specific hobby.

Nowadays, individual agents like Hermes Agent and Openclaw, when running well-designed LLM models, along with well-written set goals and context skills, can achieve desired results in a single step.

If we choose the right models/tools and skills, and write down the skills well, for example:

Let's say you're managing customer personas specifically for support customers who have particular interests. You'd then create context skills for them, like `/gamedev-persona-tall-67inches`.

Doing something like this really helps make the agents useful.

After analyzing the agents, it became clear that they were producing real results.

The correct way to proceed with automation is to validate it first, as the startup/YC community says: manual work works best. Find something that works first before double-down to automation and scaling.

Once we can automate it, we continue to the next step.

This is called a Automate Skill Chain, used to quickly produce new products, divided into areas such as Build, Sales, and Content.

Create cause & effect chain for agent
Focus on distribution with the specific success metrics number. 

Test multiple format of products, features, pages, and content, whether it's microsites, microniche youtube content, or dividing it into content pillars.

For posting on websites, including video clips, find a format that works quickly. Starting with organic content is even better. Actually, we should have agents test many different formats, including UGC (User Generated Content).

Then, collect feedback, and move on to the next step based on market reactions. Try to capture as much data as possible from emails, DMs, meetings, messages, and like reactions, and then use this data to create context for each persona.

Clean up your data to make it easier to use and reuse the JSON structure for business logic, pricing rules, and personas.

Or, if we want the Agents to have real depth and unique skills that set them apart from others, we could also apply game development knowledge. For example, since I was heavily involved with UE5 before, we could simply create AI Agents that excel at defining variables/data structures specific to each type of game character.

For example, if you want to create agents/game systems with unique capabilities, you should create variables that contain integers, booleans, and vectors, store them as arrays, and keep that data in a code file, folder, or other suitable location so the agents can easily search for it. Create a good function blueprint as well.

There might be some timeline animation curve features available on Unreal Engine 5 if you want unique results. I think this is where strategic flexibility with Agents comes in, leading to better results than focusing solely on specific actions. For example, Agents built for enterprises might have a float range for action speed, such as 0.0-1.0.

How would other variables affect the action – the acceleration/rate of change –?

This can also be applied to business applications like sales and marketing. It has an impact on everything, such as how quickly an agent can follow up, and what's best suited for each individual.

Creating Agents is like creating a Game Mode; it defines the game's environment, such as the number of teams, roles, characters, and how the game end. It also specifies which Player Controllers will control which Player Characters and what PlayerState values ​​will be applied.

Attack power, speed, health (HP), poison status, and the consequences of mistakes.

In short, we're essentially creating a good game, and if we want good AI agents, we just need to create a good game system.

Once agents create products, they test them in the market -> collect data and give it to the agents. We then run a feedback loop back to the agents, and humans continuously reinforce the strategy, tastes, and trust. 

That's how we truly build an AI-Native Organization With Game Dev Concept.
