### Allowed Deliverables (CONSTRAIN IDEAS TO THESE)

```
Sunder can ONLY deliver:
1. Document extraction (PDFs, images → structured data)
2. Document reconciliation (matching doc type A to doc type B)
3. Discrepancy/exception flagging (auto-catch errors)
4. Excel report generation (with formulas, formatted)
5. Validation rules (cross-checks, business logic)

For retailers specifically:
- Extract: Supplier invoices, delivery orders, packing lists (any format)
- Reconcile: Invoices ↔ POs, DOs ↔ invoices (3-way match)
- Flag: Quantity mismatches, price discrepancies, missing items
- Report: Reconciliation summaries, exception lists, payment approval queues

DO NOT suggest:
- General "AI will help" vague ideas
- CRM/sales features
- POS/inventory management
- Anything without a document component
```

### What Sunder Does (Value Prop)

```
Automates supplier invoice reconciliation for retailers:
- Extracts data from any supplier's invoice format (PDF, Excel, scans)
- Matches invoices to POs and delivery orders automatically
- Flags discrepancies before payment approval
- Catches overpayments, quantity errors, price mismatches
```
