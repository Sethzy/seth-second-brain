## The Problem

Claude Code has genuinely changed how I work. But there's this one thing that drives me crazy...

**Every day, I have to explain the same exact things to claude code.**

Examples of repeated explanations:
- Coding style and preferences
- "The user service connects to Postgres, not MySQL."
- "Don't refactor that function—I know it looks ugly but there's a reason it's like that."

**The cycle:**
1. Claude writes great code
2. Close the session
3. It forgets everything
4. Next day = Groundhog Day. Again.

## Existing Workarounds (All Inadequate)

People have built various workarounds:
- Massive files
- Copy pasting context at the start of every prompt
- Maintaining "memory" documents that feel like the agent is never looking at them

## The Solution: Supermemory Plugin

After success with previous projects, the team knew they were in the right spot to solve this problem.

So they built Supermemory for Claude Code.

## Background & Context

The Supermemory plugin solves the fundamental problem of Claude Code forgetting context between sessions by providing persistent memory that carries over across conversations.
