## Setup Instructions

### Step 1: Install OpenClaw

**URL:** https://openclaw.ai/
**Action:** Follow installation instructions on website

### Step 2: Choose QuickStart

**After launching OpenClaw:** Select `QuickStart`
**Purpose:** Walks through minimal configuration required

### Step 3: Select Moonshot AI as Model / Auth Provider

**In configuration:** Choose `Moonshot AI` as provider
**Options available:** 3 (detailed below)

## Three Billing Options

### Option 1: Kimi Code Plan

**Billing model:**
- Fixed monthly subscription
- Includes token usage cap

**Recommended if:**
- You're new to OpenClaw
- You want predictable costs
- You prefer fast, frictionless setup

**Setup:**
1. Select `Kimi Code API Key` in OpenClaw
2. Go to https://www.kimi.com/code/console
3. Create API Key and copy it
4. Paste into OpenClaw

**Pro tip:** "Start with a Kimi Code Plan. If you run out of quota, you can upgrade or switch to Kimi API anytime."

---

### Option 2: Kimi API (.ai) - International

**Billing model:**
- Pay only for tokens you use
- No monthly cap
- USD billing

**Recommended if:**
- You've exhausted Kimi Code Plan quota
- You want unlimited, long-running agent usage
- You're outside mainland China

**Setup:**
1. Select `Kimi API key (.ai)` in OpenClaw
2. Go to https://platform.moonshot.ai/console/api-keys
3. Create and copy API Key
4. Paste into OpenClaw

---

### Option 3: Kimi API (.cn) - Mainland China

**Billing model:**
- Pay only for tokens you use
- No monthly cap
- RMB billing (mainland China platform)

**Recommended if:**
- Your balance is on mainland China platform
- You operate primarily in China

**Setup:**
1. Select `Kimi API key (.cn)` in OpenClaw
2. Go to https://platform.moonshot.cn/
3. Create and copy API Key
4. Paste into OpenClaw

## Pricing Options Deep Dive

### Kimi Code Plan Economics

**Model:** Subscription + cap
**Benefits:**
- Predictable monthly cost
- No surprise bills
- Good for budgeting

**Limitations:**
- Hit cap = agent stops or upgrade required
- Not ideal for 24/7 heavy usage

**Target user:** Small projects, testing, intermittent use

### Kimi API Pay-Per-Token

**Model:** Usage-based
**Benefits:**
- No caps
- Only pay for what you use
- Scales to any volume

**Risks:**
- Unpredictable costs (runaway agent = big bill)
- Requires monitoring

**Target user:** Production deployments, 24/7 agents, high-volume

### .ai vs .cn Split

**Reason for split:**
- Different regulatory environments
- Separate payment systems (USD credit cards vs Alipay/WeChat Pay)
- Tax/compliance requirements

**User choice:** Based on where you're located and how you pay
